PRIMER NIVELL AL QUAL TREBALLEM ÉS:
CABLAT I LA TERGETA DE XARXA:
CABLE DE COURE -> ETHERNET /CABLE UTP CAT 5E/6/7 RJ45
F.O. -> MONOMDE/MULTIMODE
ONES ELECTROMAGNETIQUES -> WIRELESS -> WIFI
TARGETA ETHERNET -> DRIVER: ADREÇA MAC -> IDENTIFICA UNA TARGETA DINS D'UNA LAN (XARXA LOCAL)
TARGETA WIFI -> DRIVER: ADREÇA MAC -> IDENTIFICA LA TARGETA DE WIFI DINS DE LA LAN
AQUEST PRIMER NIVELL ÉS ES HARDWARE.
SEGON NIVELL AL AQUAL TREBALLEM ÉS
CONFIGURACIÓ IP -> TREBALLAR AMB ADRECES IP
LES ADRECES IP IDENTIFIQUEN UN EQUIP DINS DE LA XARXA
LES ADRECES IP PODEN SER UTILITZADES A NIVELL DE XARXA LOCAL PERÒ TAMBÉ A NIVELL D'INTERNET.
PER PODER TREBALLAR AMB IP CAL TENIR INSTALAT EL SOFTWARE DEL PROTOCOL IP DINS DE L'EQUIP.
AQUEST NIVELL ÉS MÉS SOFTWARE.
DINS DE LA XARXA LOCAL ÉS NORMAL QUE UNA ADREÇA MAC CORRESPONGUI A UNA ADREÇA IP.
EL SOFTWARE DEL PROTOCOL IP ESTÀ AVUI INCORPORAT AL SISTEMA OPERATIU.
UN TERCER NIVELL ÉS:
CONFIGURACIÓ DEL PROTOCOL TCP O UDP -> ESTÀ RELACIONAT AMB EL CONCEPTE DE PORT TCP O UDP
ELS PORTS TCP O UDP SÓN LÒGICS,O SIGUI SÓN NÚMEROS.
ELS PORTS IDENTIFIQUEN PROGRAMES DINS DE L'ORDINADOR.
PER PODER TREBALLAR AMB TCP I UDP CAL INSTALAR EL SOFTWARE DEL PROTOCOL TCP I EL PROTOCOL UDP DINS DE L'ORDINADOR. AVUI DIA ESTÀ DINS DEL SISTEMA OPERATIU.
UN DARRER NIVELL ÉS 
APLICACIONS -> CLIENTS I SERVIDOR
UNSOFTWARE -> S'INSTALEN O DESINSTALEN SEGONS LES NOSTRES NECESSITATS 
NO FORMEN PART DEL SISTEMA OPERATIU 
APLICACIONS ASSOCIADES A PROTOCOLS -> HTTP, DNS, DHCP, SMTP, POP3, SSH, FTP, SMB,...
PODEN EXISTIR CENTENARS DE PROTOCOLS A DIFERENTS APLICACIONS 
AQUESTS PROTOCOLS REBEN EL NOM DE PROTOCOLS DE NIVELL D'APLICACIÓ.


NOMS DELS NIVELLS
SI PARLO D'APLICACIONS EN XARXES S'UTILITZA LA NOMENCLATURA SEGUENT: NIVELL D'APLICACIÓ.
SI PARLEM DE TCP I UDP. NIVELL DE TRANSPORT. DIEM QUE TCP I UDP SÓN PROTOCOLS DE TRANSPORT.
SI PARLEM D'IP -> NIVELL DE XARXA. IP ÉS UN PROTOCOL DE NIVELL DE XARXA O TAMBÉ DE NIVELL INTERNET. SI PARLEM DE CABLES I TARGETES -> NIVELL D'ENLLAÇ DE DADES.
A AQUEST NIVELL PARLEM DE NORMES DE COMUNICACIÓ MÉS QUE DE PROTOCOLS.
 




A CADA NIVELL DE XARXA TREBALLEM AMB AGRUPACIOONS DE BYTES QUE TENEN NOMS DIFERENTS.
A NIVELL D'APLICACIÓ PARLEM DE MISSATGES 
A NIVELL DE TRANSPORT:
	TCP -> SEGMENTS TCP
	UDP -> DATAGRAMES UDP
A NIVELL DE XARXA PARLEM DE PAQUETS I MÉS CONCRETAMENT DE PAQUETS IP
A NIVELL D'ENLLAÇ DE DADES PARLEM DE TRAMES (PER EXEMPLE TRAMES ETHERNET) 
UNA ADREÇA MAC -> ES UNA ADREÇA DE NIVELL D'ENLLAÇ -> IDENTIFICA UNA TARGETA DE XARXA
ADREÇA IP -> ADREÇA DE NIVELL DE XARXA -> IDENTIFICA UN EQUIP(ORDINADOR, ROUTER, IMPRESSORA)
PORT TCP O UDP -> ADREÇA DE NIVELL DE TRANSPORT -> IDENTIFICA UN PROGRAMA DINS D'UN EQUIP.


UN MISSATEG D'APLICACIÓ ON HTTP, DNS... FORMARÁ PART D'UN SEGMENT TCP O DATAGRAMA UDP QUAN VIATGE PER LA XARXA (LOCAL O INTERNET). 
PER EXEMPLE UN SEGMENT TCP = UNES DADES PRÒPIES DE TCP + MISSATGE HTTP.
UN DATAGRAMA UDP = UNES DADES PRÒPIES D'UDP + MISSATGE DNS.
DIREM QUE EL MISSATGE DNS ESTÀ ENCAPSULAT DINS D'UN DATAGRAMA UDP O UN MISSATGE HTTP ESTÀ ENCAPSULAT DINS D'UN SEGMENT TCP.

IGUALMENT UN SEGMENT TCP S'ENCAPSULA DINS D'UN PAQUET IP, UN DATAGRAMA UDP S'ENCAPSULA DINS D'UN PAQUET IP I FINALMENT UN PAQUET S'ENCAPSULA DINS D'UNA TRAMA ETHERNET.
